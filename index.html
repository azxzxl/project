[*] Comandos para realizar varreduras [*]

nmap -vvv -sS --open -Pn -p- -T 5 |IP|

  • esse comando realiza uma varredura intensiva e detalhada de todas as portas TCP abertas em uma máquina, fornecendo informações úteis para a identificação de possíveis vulnerabilidades.

    • nmap -vvv -sV -sC -O -p -T 5 -A --script vuln -Pn |PORTS| |IP|

  • o comando acima é usado para realizar uma varredura mais completa e detalhada em um alvo, incluindo detecção de versão de serviços, detecção de sistema operacional, teste de vulnerabilidades e execução de scripts personalizados

    • nmap -sV --script=vuln

  • Ao executar esse comando, o Nmap realizará uma varredura de portas no alvo especificado e, em seguida, executará o script de detecção de vulnerabilidades "vuln" nos serviços encontrados nas portas abertas. O Nmap tentará identificar vulnerabilidades conhecidas com base nas versões de serviço detectadas e fornecerá um relatório com os resultados.

    • nikto -h |alvo|

  • Substitua pelo endereço IP ou nome de domínio da aplicação web que você deseja varrer. Esse comando realizará uma varredura básica em busca de vulnerabilidades conhecidas.

    • nikto -h |alvo| -o |arquivo_de_saida|

  • Substitua |alvo| pelo endereço IP ou nome de domínio do site e |arquivo_de_saida| pelo caminho e nome de arquivo onde você deseja salvar os resultados da varredura. Essa opção é útil para revisar os resultados posteriormente ou compartilhá-los com outras pessoas.

    • [*] Ferramentas para port scan [*]

    nc -vnz |IP| 80

  • Escaneia a porta 80 do IP

    • nc -vz www.exemplo.com 80

  • escaneia a porta 80 do site

    • nc -vnz |IP| 20-1024

  • escaneia da porta 20 à 1024

    dnsenum site.com

  • Enumeraçao de DNS, brute force etc.

    • dnsrecon site.com

  • Apresenta as possíveis vulnerabilidades do domínio

    • fierce -dns site.com

  • Apresenta as possíveis vulnerabilidades do domínio com bruteforce

    • [*] Ferramentas para ataques de força bruta [*]

    A seguir algumas formas de realizar um brute force:

    hydra -l |nome_de_usuário| -P |caminho_para_lista_de_senhas| ssh://|alvo|

  • Substitua |nome_de_usuário| pelo nome de usuário alvo, |caminho_para_lista_de_senhas| pelo caminho para um arquivo contendo uma lista de senhas que você deseja testar e |alvo| pelo endereço IP ou nome de domínio do servidor SSH.

    • hydra -l |nome_de_usuário| -P |caminho_para_lista_de_senhas| mysql://|alvo|

  • Substitua |nome_de_usuário| pelo nome de usuário alvo, |caminho_para_lista_de_senhas| pelo caminho para um arquivo contendo uma lista de senhas que você deseja testar e |alvo| pelo endereço IP ou nome de domínio do servidor MySQL.
